# **COMPILE**

This command is used when the user says:

```text
COMPILE
```

The agent must compile the source Agent OS Markdown into a minimal, lazy-loading Markdown OS under `os/min`.

---

## **Purpose**

Build a hyper-optimized Agent OS distribution for an LLM target audience that minimizes token consumption while preserving command correctness.

The compiled OS is designed for expert LLM context engineering: boot loads only the smallest possible operating contract, command documents are loaded only on exact invocation, and stale generated files are removed from the compiled output.

The optimization goal is functional equivalence at minimum token cost. The compiler should treat every generated Markdown character as context budget that must justify itself.

---

## **Input**

- `source_root`: optional path; default `os`.
- `output_root`: optional path; default `os/min`.

Both paths must remain inside the repository. `output_root` must be inside `source_root`.

---

## **Side Effects**

This command may:

- Create directories under `os/min`.
- Create or replace `.md` files under `os/min`.
- Remove stale files and empty directories under `os/min` that are not part of the current compiled output.

This command must not:

- Modify files outside `os/min`.
- Delete source files under `os` outside `os/min`.
- Stage, commit, reset, or otherwise alter Git state.
- Call external services or require network access.

---

## **Compilation Rules**

When invoked, the agent must:

1. Load the source Agent OS documents needed to understand boot, command semantics, invariants, and available commands.
2. Discover command documents only from direct child Markdown files matching `os/commands/*.md`.
3. Exclude `os/commands/Invariants.md` and do not recursively scan nested command directories such as `os/commands/git`.
4. Generate `os/min/BOOT.md` as the only required boot file.
5. Embed the minimal identity, mission, safety, invocation, side-effect, and lazy-loading rules directly in `os/min/BOOT.md`.
6. Embed a compact command registry in `os/min/BOOT.md` with trigger phrase, purpose, path, and side-effect scope for each compiled command.
7. Generate one minimal command file per invokable command under `os/min/commands`, except `BOOT`, which is represented by `os/min/BOOT.md`.
8. Make each compiled command self-sufficient enough to execute after it is loaded, without requiring the full source command model.
9. Prefer short imperative rules, dense command tables, and explicit decision logic over explanatory prose.
10. Optimize for LLM parsing and token economy: remove redundancy, examples, tutorial text, decorative prose, repeated headings, and non-operational commentary.
11. Use compact Markdown structures that are easy for an LLM to follow, including tables, terse lists, explicit condition/action rules, and stable path names.
12. Preserve source-truth semantics while changing wording, layout, and file boundaries whenever that reduces token load without reducing behavior.
13. Maximize lazy and dynamic loading: boot must contain only dispatch and global rules; command-specific details must stay in command files loaded only after exact trigger match.
14. Ensure `os/min/BOOT.md` instructs agents not to eagerly load command files; they must load a command file only after exact trigger match.
15. Ensure compiled commands may cache loaded command files and discovery results in working memory for the current session, invalidating that cache if relevant files change or the user reruns `COMPILE`.
16. Remove all stale files under `os/min` that are not generated by the current compilation.

---

## **Generated File Requirements**

The compiled output must include:

```text
os/min/BOOT.md
os/min/commands/<direct-command-name>.md
```

The command file set must be derived dynamically from direct child command documents under `os/commands/*.md`, excluding `Invariants.md`. Do not hard-code a default command list.

---

## **Output**

Use this shape:

```text
COMPILE complete.

Generated:
- <path>

Removed:
- <path or "none">

Boot:
- /os/min/BOOT.md

Notes:
- <short note about lazy loading or cache behavior>
```

---

## **Failure Behavior**

If the command cannot run, the agent must:

- Report the missing source document, invalid path, denied permission, or write failure.
- Avoid modifying files outside `os/min`.
- Avoid inventing command behavior that is not present in source command documents.
